# New Hono Project
 
Scaffolds a production-ready Hono app on Cloudflare Workers with:
- **Hono** (latest, ~4.x) — ultrafast web framework
- **Cloudflare Workers** — runtime + deployment target
- **D1** — Cloudflare's serverless SQLite database
- **Drizzle ORM** (latest, ~0.45.x) — type-safe SQL ORM with D1 adapter
- **TypeScript 5.x** — latest compatible version
- **Wrangler** — CLI for dev, deploy, and D1 management

## Package Manager
- **Always use pnpm** — never npm or yarn
- All install instructions must use `pnpm add`, `pnpm install`, `pnpm dlx`
- Lock file: `pnpm-lock.yaml`

---
 
## 1. Determine Project Name
 
Ask the user for a project name if not provided. Use `kebab-case`. Default: `my-hono-app`.
 
---
 
## 2. Scaffold with create-hono
 
```bash
npm create hono@latest <project-name> -- --template cloudflare-workers
cd <project-name>
npm install
```
 
> `create-hono` bootstraps the Workers entry point, `wrangler.jsonc`, and `tsconfig.json` automatically. Always use it — don't hand-write the scaffold.
 
---
 
## 3. Install Drizzle + D1 Dependencies
 
```bash
npm install drizzle-orm
npm install --save-dev drizzle-kit
```
 
> Do NOT install `@libsql/client` or `better-sqlite3` — D1 uses the Workers runtime binding, not a local client.
 
---
 
## 4. Configure wrangler.jsonc
 
Add the D1 binding and enable required flags. Edit `wrangler.jsonc`:
 
```jsonc
{
  "name": "<project-name>",
  "main": "src/index.ts",
  "compatibility_date": "<today>",          // use today's date: YYYY-MM-DD
  "compatibility_flags": ["nodejs_compat"],
  "observability": {
    "enabled": true,
    "head_sampling_rate": 1
  },
  "d1_databases": [
    {
      "binding": "DB",                       // must match Env interface key
      "database_name": "<project-name>-db",
      "database_id": "PLACEHOLDER"           // replaced after `wrangler d1 create`
    }
  ]
}
```
 
> **Never hardcode secrets** in `wrangler.jsonc`. Use `wrangler secret put` for sensitive values.
 
---
 
## 5. Regenerate Env Types
 
After editing `wrangler.jsonc`, always run:
 
```bash
npx wrangler types
```
 
This regenerates `worker-configuration.d.ts` so `Env` reflects the actual D1 binding. **Never hand-write the `Env` interface.**
 
---
 
## 6. Set Up Drizzle Schema
 
Create `src/db/schema.ts`:
 
```ts
import { integer, sqliteTable, text } from "drizzle-orm/sqlite-core";
 
// Example table — replace with your domain model
export const users = sqliteTable("users", {
  id: integer("id", { mode: "number" }).primaryKey({ autoIncrement: true }),
  name: text("name").notNull(),
  email: text("email").notNull().unique(),
  createdAt: integer("created_at", { mode: "timestamp" })
    .$defaultFn(() => new Date())
    .notNull(),
});
 
export type User = typeof users.$inferSelect;
export type NewUser = typeof users.$inferInsert;
```
 
---
 
## 7. Set Up Drizzle Client Helper
 
Create `src/db/index.ts`:
 
```ts
import { drizzle } from "drizzle-orm/d1";
import * as schema from "./schema";
 
export function createDb(d1: D1Database) {
  return drizzle(d1, { schema });
}
```
 
> Pass the D1 binding per-request from `env.DB` — never store it in module-level state.
 
---
 
## 8. Configure drizzle.config.ts
 
Create `drizzle.config.ts` at the project root:
 
```ts
import { defineConfig } from "drizzle-kit";
 
export default defineConfig({
  dialect: "sqlite",
  schema: "./src/db/schema.ts",
  out: "./drizzle",
  driver: "d1-http",       // required for Drizzle Kit + D1
  dbCredentials: {
    accountId: process.env.CLOUDFLARE_ACCOUNT_ID!,
    databaseId: process.env.CLOUDFLARE_D1_DATABASE_ID!,
    token: process.env.CLOUDFLARE_API_TOKEN!,
  },
});
```
 
---
 
## 9. Write the Hono Entry Point
 
Replace `src/index.ts` with:
 
```ts
import { Hono } from "hono";
import { createDb } from "./db";
import { users } from "./db/schema";
 
type Bindings = Env; // generated by `wrangler types`
 
const app = new Hono<{ Bindings: Bindings }>();
 
app.get("/", (c) => c.json({ message: "Hello Hono!" }));
 
// Example: list users
app.get("/users", async (c) => {
  const db = createDb(c.env.DB);
  const result = await db.select().from(users);
  return c.json(result);
});
 
// Example: create user
app.post("/users", async (c) => {
  const body = await c.req.json<{ name: string; email: string }>();
  const db = createDb(c.env.DB);
  const [user] = await db.insert(users).values(body).returning();
  return c.json(user, 201);
});
 
export default app;
```
 
**Workers best-practice notes:**
- `Hono<{ Bindings: Bindings }>` types all `c.env.*` bindings correctly
- `createDb(c.env.DB)` is called per-request — no module-level mutable state
- Use `ctx.waitUntil()` (available via `c.executionCtx.waitUntil()`) for post-response async work
---
 
## 10. Add package.json Scripts
 
Add these scripts to `package.json`:
 
```json
{
  "scripts": {
    "dev": "wrangler dev",
    "deploy": "wrangler deploy",
    "types": "wrangler types",
    "db:generate": "drizzle-kit generate",
    "db:migrate:local": "wrangler d1 migrations apply <project-name>-db --local",
    "db:migrate:remote": "wrangler d1 migrations apply <project-name>-db --remote",
    "db:studio": "drizzle-kit studio"
  }
}
```
 
---
 
## 11. Create the D1 Database (One-Time)
 
```bash
npx wrangler d1 create <project-name>-db
```
 
Copy the printed `database_id` into `wrangler.jsonc` replacing `"PLACEHOLDER"`.
 
---
 
## 12. Generate & Apply Migrations
 
```bash
# Generate SQL migration from schema
npm run db:generate
 
# Apply locally (dev)
npm run db:migrate:local
 
# Apply to production
npm run db:migrate:remote
```
 
---
 
## 13. TypeScript Config
 
`create-hono` generates a working `tsconfig.json`. Verify it includes:
 
```json
{
  "compilerOptions": {
    "target": "ESNext",
    "module": "ESNext",
    "moduleResolution": "Bundler",
    "lib": ["ESNext"],
    "strict": true,
    "types": ["@cloudflare/workers-types"]
  }
}
```
 
> TypeScript 5.x is required. `moduleResolution: "Bundler"` is the modern setting — do not use `"node"` or `"node16"`.
 
---
 
## 14. Dev & Deploy
 
```bash
# Local dev with D1 local SQLite
npm run dev
 
# Deploy to Cloudflare Workers
npm run deploy
```
